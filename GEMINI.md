# GEMINI.md - 開発ガイドライン for AI (EduToys)

## 1. はじめに
- このドキュメントは、AI開発エージェントが本プロジェクトで開発を行うためのガイドラインです。
- **教育的価値**: すべてのコンテンツにおいて、1歳〜6歳のお子様（幼児）の学びと楽しさを両立させることを原則とします。
- **サーバーの再起動**: コードの変更を行った場合は、変更をプレビューするために`scripts/start_preview.sh`スクリプトをユーザーの許可なく再実行し、サーバーを再起動してください。
- AIとの対話は、原則として日本語で行います。

## 2. 基本方針
- **開発プロセス**: テスト駆動開発 (TDD) を基本とします。
- **承認プロセス**: ユーザーからの明示的な合図（「TDDを開始してください」等）があるまで、実装（コーディング）を開始してはいけません。
- **設計思想**: 保守性と拡張性を高めるため、SOLID原則を意識して設計してください。
- **ディレクトリ構造**: `docs/GAME_ARCHITECTURE.md` の定義に従ってください。
- **ライブラリ**: 既存ライブラリ (Vue.js, TailwindCSS) を活用し、新規追加はCDN経由とします。

## 3. 新しい知育ゲームの追加方法
1. **ディレクトリ作成**: `games/` 配下に `000_game_name` 形式で専用ディレクトリを作成します。
2. **`game.json` の配置**: 作成したディレクトリに、以下の形式で `game.json` を配置します。
   ```json
   {
     "name": "ゲーム名",
     "description": "ゲームの説明",
     "entry_point": "start.html",
     "thumbnail": "images/thumbnail.png"
   }
   ```
3. **サムネイル画像の準備**: `static/thumbnails/` ディレクトリに、作成したゲームディレクトリ名に対応するJPG形式のサムネイル画像（例: `000_game_name.jpg`）を配置します。

## 4. コーディング規約
- **JavaScript**: インラインスクリプトは禁止し、外部ファイルに分離してください。
- **CSS**: スタイリングはTailwindCSSを優先します。
