# 001_count_tap (かぞえてタップ) 実装プラン

## 1. 概要
「かず」カテゴリの最初のゲーム。画面上にランダムに散らばったオブジェクトをタップし、すべてタップし終えるとクリアになる、2歳〜向けのシンプルなゲームです。
PixiJSでのオブジェクト配置、GSAPでのタップ時のアニメーション、そしてHowler.jsでの音声再生といった基本アーキテクチャの動作検証も兼ねます。

## 2. ランダム性とリプレイ性の確保（ユーザー要望）
毎回同じ画面にならないよう、プレイのたびに以下の要素をランダム化し、飽きが来ないようにします。
1. **オブジェクトの数**: 3個 〜 10個 の間でランダム。
2. **オブジェクトの種類（テーマ）**: りんご、くるま、ほし、おにぎり など、プレイごとにテーマをランダムに抽選（画像はSVGで用意）。
3. **配置座標**: PixiJSのCanvas領域内で、オブジェクト同士が重なりすぎないようにランダムなX/Y座標を計算して配置。
4. **大きさ・角度**: オブジェクトごとにわずかなスケールの差異（0.8倍〜1.2倍）や、初期の傾き（-15度〜15度）を付与し、自然なバラツキを出す。

## 3. UIとCanvasの役割分担
*   **UIレイヤー (DOM/Vue)**:
    *   画面上部の「ぜんぶ タップしてね！」などの指示テキスト。
    *   現在のカウント数 / 全体の数（例: `3 / 5`）のテキスト表示。
    *   クリア時の「シールちょう / もういっかい」オーバーレイ。
*   **ゲームレイヤー (PixiJS)**:
    *   背景の描画（パステルカラーや簡単なパターン）。
    *   ランダム配置されたオブジェクトの描画。
    *   タップ判定と「弾ける」アニメーション（GSAPを使用）。

## 4. インタラクションと演出
1.  **出現時**: オブジェクトが順番にポン、ポンと飛び出してくるアニメーション（GSAP `back.out` イージング等）。
2.  **タップ時**:
    *   オブジェクトが少し膨らんでから（弾けて）消えるアニメーション。
    *   キラキラする星型のパーティクルを少量発生させる。
    *   効果音（「ポンッ」や「ピコッ」などの小気味よい音）を鳴らす。
    *   画面上部のカウント表示が更新され、「いち！」「に！」といった数を数えるボイス（四国めたん）を鳴らす。
3.  **クリア時**:
    *   最後のオブジェクトを消すと、「よくできました！」のボイスとクリアジングルが鳴る。
    *   大きな「はなまる」がCanvas上に描画される。

## 5. 必要なアセット（生成・配置するもの）
*   **画像 (SVG)**:
    *   `games/001_count_tap/images/apple.svg`
    *   `games/001_count_tap/images/car.svg`
    *   `games/001_count_tap/images/star.svg`
    *   `static/images/hanamaru.svg` (共通のはなまる)
*   **音声 (Voicevox / 既存素材)**:
    *   「いち」〜「じゅう」、「よくできました」のボイス（VOICEVOX四国めたん）
    *   タップ音、クリア音（`static/sounds/` 内の既存ファイルから選定）

## 6. 実装ステップ
1.  **ディレクトリと基本ファイルの作成**: `games/001_count_tap/` 配下に `game.json`, `start.html`, `js/game.js` などを配置。
2.  **SVG画像の準備**: りんご等のプレースホルダーSVGを作成。生成する画像は日本の平成〜令和時代のアニメ調で可愛らしくシンプルなものとする。
3.  **PixiJSの初期化とランダム配置ロジック**: Canvas上に指定数・指定種類のオブジェクトを重ならないように配置するロジックを実装。
4.  **タップ判定とGSAPアニメーション**: タップ時にオブジェクトが消える演出を実装。
5.  **Vue.jsとの連携と状態管理**: タップした数をDOMレイヤーのスコアボードに反映し、全部消えたらクリア処理を呼び出す。
6.  **音声の組み込み**: タップ音とVOICEVOXで生成したカウントボイスを紐付け。
