# 003_hiragana_suisui（ひらがな すいすい）実装プラン

## 1. 概要

「もじ」カテゴリの最初のゲーム。大きなひらがなのガイド線を、指やマウスで**やさしく なぞって 完成させる** 3歳〜向けの文字導入ゲームです。

厳密な手書き認識ではなく、**太いおみちの上を気持ちよくたどる体験**を重視します。幼児が「もじの形」と「読み」を楽しく結び付けられるよう、音声ガイド、きらきら演出、はなまる演出を中心に構成します。

**GAME_IDEAS.md との対応**: `ひらがな基礎 | 3歳〜 | SVGパス追従、書き順判定 | 書き終えた後に文字が跳ねる`

## 2. ねらいと初期スコープ

### 2.1 学習目標
- ひらがなの**形を目で覚える**
- 音声と文字を結び付けて**読みの入口**を作る
- 指先を使って、**なぞる・進む・止まる**の基本操作に慣れる

### 2.2 初期リリースの範囲
- 出題候補は **animCJK 由来の全ひらがなデータ** を使う
- 1プレイごとに候補の中から **3文字前後をランダム出題** し、すべて終えたらクリア
- 判定は「正しい線の近くを進めたか」を見る**やさしいガイド追従型**
- 書き順は扱うが、少し外れても成功しやすいよう**広めの判定幅**を持たせる
- 初回は同じ文字が続きすぎないよう、ランダム抽選に軽いバランス制御を入れる

### 2.3 今回はやらないこと
- 自由手書き文字の高精度認識
- はらい・とめ・はねの厳密判定
- 1回のプレイで全ひらがなを通しで遊ぶ長時間モード
- カタカナモードの同時実装

## 3. ゲームデザイン

### 3.1 基本ルール
- 画面中央に、今遊ぶひらがなの**大きな見本**を表示する
- 各プレイでは **全ひらがな候補の中からランダムに選ばれた3文字前後** を順番に遊ぶ
- 文字ごとに、1画ずつ**光るガイド線**が表示される
- 幼児はスタートの丸から線をなぞる
- ガイドの上を十分進めると、その1画が完成して色づく
- すべての画を終えると、文字が「ぴょこん」と弾み、音声で文字を読み上げる
- 3文字前後を終えると、クリアオーバーレイ → シール選択へ進む

### 3.2 出題文字

- ベースデータは `static/characters/animcjk/hiragana.json` に置いた **全ひらがな** を使う
- 1プレイではその中から3文字前後を抽選する
- 難易度設計は、将来的に以下のようなグループで重み付けできるようにする

| グループ | 例 | ねらい |
|:---|:---|:---|
| やさしい直線・短画中心 | い、く、し、こ | 最初の成功体験を作りやすい |
| 曲線中心 | う、つ、て、の | 指でたどる感覚を育てやすい |
| 交差や複合形 | あ、ぬ、め、を | 少し複雑な形にも慣れていく |
| 小書きかな | ぁ、ぃ、ゃ、ゅ、ょ | 将来の拡張に備えた候補 |

### 3.3 進行フロー
1. プレイ開始時に、全ひらがな候補からその回の **3文字前後** を抽選
2. 「あ を すいすい なぞってね！」のように、現在文字の音声ガイド
3. 現在の1画だけを強調表示
4. なぞり成功で、星・音・色変化
5. 1文字完了で文字読み上げ + はなまる風フィードバック
6. 次の文字へ自動遷移
7. 抽選された3文字前後を終えたらゲームクリア

### 3.4 失敗時の扱い
- 線から大きく外れたら、その場で赤くせず、**ガイドをふわっと再点灯**
- 音声や強い否定は使わず、「もういちど なぞってみようね」程度のやさしい案内に留める
- 画の途中までは進捗を保持せず、必要ならその画だけ最初からやり直す

## 4. UIレイアウト

```text
┌───────────────────────────────────────────┐
│ [◀ もどる]              もじ 1 / 3         │  ← DOM (ヘッダー)
├───────────────────────────────────────────┤
│   「あ を すいすい なぞってね！」            │  ← DOM (指示テキスト)
│                                           │
│        [ <canvas> PixiJS ]                │  ← ガイド線・なぞり・演出
│                                           │
│        [ おてほんの もじ ]                 │
├───────────────────────────────────────────┤
│     [終了オーバーレイ (hidden)]             │
└───────────────────────────────────────────┘
```

### 4.1 レスポンシブ方針
- **横置きタブレット**: 文字を大きく中央配置、上部に進捗表示
- **縦持ちスマホ**: ヘッダーを圧縮しつつ、文字表示領域を最優先
- ガイド文字は常に大きく見せるため、Canvas 内の有効領域は上下左右に十分な余白を取る

## 5. PixiJS 実装詳細

### 5.1 描画レイヤー
1. **背景レイヤー**: やわらかい色鉛筆・おえかき帳風の背景画像
2. **おてほんレイヤー**: うすいグレーの文字シルエット
3. **ガイドレイヤー**: 現在なぞる画の太い発光ライン、スタート丸、ゴール星
4. **なぞりレイヤー**: 実際に通った軌跡、進捗に応じた色変化
5. **エフェクトレイヤー**: キラキラ、はなまる、完成時のバウンド

### 5.2 文字データの持ち方
- 文字ごとに、**画単位のパス定義**を持つ
- データソースは `static/characters/animcjk/hiragana.json`
- animCJK 由来の `guidePath` / `outlinePath` を、ゲーム側で扱いやすい形へ正規化して使う
- 1文字ごとに以下の情報を保持する:
  - `id`（`hiragana-3042` のような一意ID）
  - `label`（`あ` など）
  - `strokes`（画ごとのパス情報）
  - `voice`（対応する音声ファイル）
  - `guideText`（開始時メッセージ）
  - `difficultyGroup`（抽選バランス調整用）

想定データ例:

```javascript
{
  id: 'hiragana-3042',
  label: 'あ',
  guideText: 'あ を すいすい なぞってね！',
  strokes: [
    { guidePath: 'M...', outlinePath: 'M...' },
    { guidePath: 'M...', outlinePath: 'M...' },
    { guidePath: 'M...', outlinePath: 'M...' }
  ],
  difficultyGroup: 'curve-mixed'
}
```

### 5.3 なぞり判定
- `pointerdown` は**スタート丸の近く**でのみ受付
- `pointermove` で現在の点がガイド線の近傍にあるかを判定
- 各画は複数の**チェックポイント**に分割し、順番に通過したら進捗を加算
- 最終チェックポイントまで進むとその画をクリア
- 判定は厳しすぎないよう、線幅に加えて**余白バッファ**を設ける

### 5.4 演出
- なぞっている間: 軌跡がきらっと光る
- 1画クリア時: 小さな星がパッと出る
- 1文字クリア時: 文字全体が色づいて、`GSAP` で「ぴょこん」と弾む
- 全クリア時: はなまる + クリア音 + ごほうびシール

## 6. 音声・SE

### 6.1 VOICEVOX 音声（四国めたん / Speaker ID: 2）
`games/003_hiragana_suisui/voices/` に配置:

| ファイル名 | テキスト | 用途 |
|:---|:---|:---|
| `guide_*.mp3` | 〇〇 を すいすい なぞってね！ | 出題された文字ごとの開始ガイド |
| `letter_*.mp3` | 〇〇！ | 出題された文字の完了時 |
| `clear.mp3` | ひらがな できたね！ すごーい！ | 全クリア時 |

- 初回実装では、全ひらがな分の音声ファイルを用意するか、または生成済み音声キャッシュ戦略を設ける

### 6.2 効果音（既存素材から選定）
- BGM: やわらかめの既存BGMを1曲使用
- なぞり開始: 軽いポヨン音
- 1画クリア: きらっとした短音
- 1文字クリア: 上向きのグロッケン系
- 全クリア: ジャジャーン系 + シール獲得音

## 10. 実装ステップ

1. **プラン確定**: 初期スコープを「全ひらがな候補」「1プレイ3文字前後」「やさしいなぞり判定」に固定
2. **テスト追加**: 003 の登録・アセット・文字データ定義に関する失敗テストを追加
3. **ディレクトリ作成**: `game.json`, `start.html`, `style.css`, `game.js`, `letter-data.js` のひな形を配置
4. **文字データ作成**: `static/characters/animcjk/hiragana.json` を元に、全ひらがなの画データと表示情報を整備
5. **PixiJS コア実装**:
   - 背景表示
   - ガイド線描画
   - 入力追跡
   - 画完了 / 文字完了 / 全体クリアの状態遷移
   - 1プレイ3文字前後の抽選
6. **GSAP 演出実装**:
   - 軌跡の発光
   - 文字完了バウンド
   - はなまる表示
7. **音声統合**:
   - 文字ごとのガイド
   - 完了ボイス
   - BGM / SE
8. **アセット生成**:
   - 背景画像
   - サムネイル画像
   - 必要なら補助アイコン
9. **シール連携**:
   - 003 の報酬プールを定義
   - 共通クリアフローへ接続
10. **回帰確認**:
   - 既存テスト実行
   - プレビュー起動
   - 実機サイズでなぞりやすさを確認

## 11. 注意事項

- 厳密な書字判定よりも、**成功体験の多さ**を優先する
- 幼児向けなので、細い線・小さい当たり判定・高速入力前提のUIは避ける
- 文字は見本として十分大きく、背景より高コントラストに保つ
- SPA内のパスは `games/003_hiragana_suisui/...` と `static/...` を使う
- 終了時は `app.destroy(true, true)`, `Howler.unload()`, `gsap.killTweensOf("*")` を確実に実行する
- クレジット `VOICEVOX:四国めたん` をゲーム画面のどこかに表示する
