# 002_shape_fit（かたちぴったん）実装プラン

## 1. 概要

「かず」カテゴリの2つ目のゲーム。画面右側にカラフルな**ドット絵の図形ピース**が並び、左側にそれらと同じかたちの**ドット絵のはめこみ枠**が配置されています。幼児がピースをドラッグして正しい枠にはめると「ぴったん！」と吸い込まれる演出とともに正解になる、2歳〜向けのかたち認識ゲームです。

**GAME_IDEAS.md との対応**: `かたちあてはめ | 2歳〜 | 幾何学図形と枠の衝突判定 | 正しい枠に吸い込まれる`

## 2. ゲームデザイン

### 2.1 基本ルール
- 画面中央〜右側にカラフルな**ドット絵図形ピース**がランダムに散らばっている
- 画面左側に同じかたちの**ドット絵のはめこみ枠**が並んでいる
- 幼児がピースをドラッグして枠に近づけると、正しいペアなら「吸い込まれる」アニメーションで枠にぴったりはまる
- 間違った枠にドラッグすると、ぶるっと揺れて元の位置に戻る
- すべての図形をはめ終えるとクリア

### 2.2 図形セット（レベルに応じて段階的に）
各ラウンドで使う図形の組み合わせをレベルごとにプリセットで用意する：

| レベル | 図形数 | 使用する図形 | 対象年齢目安 |
|:---:|:---:|:---|:---:|
| 1 | 3 | まる、さんかく、しかく | 2歳〜 |
| 2 | 4 | まる、さんかく、しかく、ほし | 2歳〜 |
| 3 | 5 | まる、さんかく、しかく、ほし、ハート | 3歳〜 |
| 4 | 6 | まる、さんかく、しかく、ほし、ハート、ひしがた | 3歳〜 |

- ゲーム開始時にレベル1から始まり、クリアするたびに次のレベルに進む
- レベル4をクリアするとゲームクリア（シール獲得）
- **リプレイ性**: 毎回、枠の表示順・ピースの初期配置・色をランダム化

### 2.3 図形の色とドット絵表現
図形ごとに固有カラーを割り当て、視覚的に区別しやすくする：
- まる: `#FF6B6B`（あか）
- さんかく: `#4ECDC4`（みどり）
- しかく: `#45B7D1`（あお）
- ほし: `#FFD93D`（きいろ）
- ハート: `#FF6B9D`（ぴんく）
- ひしがた: `#C084FC`（むらさき）

- ピース画像は**1マス感が見えるドット絵PNG**で作成し、色面を大きく、角を少し丸めた幼児向けデザインにする
- 枠画像も同じシルエットを使った**ドット絵のくぼみ表現**にし、薄いグレー (`#D9D9D9`) と内側の影で「ここに入る」ことを直感的に伝える
- ピースと枠は同一のベースシルエットから作り、見た目で迷いにくくする

## 4. PixiJS 実装詳細

### 4.1 描画レイヤー構成
1. **背景レイヤー**: パステルカラーのドット絵背景画像
2. **枠レイヤー**: 各図形の「はめ込み先」を表すドット絵スプライト
3. **ピースレイヤー**: ドラッグ可能なドット絵図形スプライト
4. **エフェクトレイヤー**: パーティクル・はなまる等

### 4.2 図形の描画方法
`PIXI.Graphics` でのベクター描画ではなく、**ドット絵PNGアセット**をスプライトとして使用する。

- 各図形ごとに `piece` と `slot` の2種類を用意する
- ベースサイズは **64x64px** で制作し、表示時は **整数倍（2x or 3x）** で拡大する
- PixiJS では `PIXI.SCALE_MODES.NEAREST` を使い、ドットの輪郭をぼかさない
- ピースは明るい本体色 + 1px相当の縁取り
- 枠は同じシルエットを薄グレー化し、内側に1段暗い影を入れて「くぼみ」に見せる

想定アセット例:

```text
pieces/circle.png
pieces/triangle.png
pieces/square.png
pieces/star.png
pieces/heart.png
pieces/diamond.png
slots/circle_slot.png
slots/triangle_slot.png
slots/square_slot.png
slots/star_slot.png
slots/heart_slot.png
slots/diamond_slot.png
```

必要に応じて、当たり判定は画像の透明部分ではなく「図形ID一致 + 中心距離」で扱い、幼児が少しずれても成功しやすい判定にする。

### 4.3 ドラッグ＆ドロップ
- `eventMode = 'static'`、`cursor = 'grab'` をピースに設定
- `pointerdown` → `pointermove` → `pointerup` でドラッグ処理
- ドラッグ中はピースを少し拡大 (`scale * 1.1`) + 影付きで「持ち上げ感」を演出
- ドロップ時に最寄りの枠との距離を判定（閾値: 枠サイズの50%以内）
- **タッチ対応**: Canvas に `touch-action: none` 設定済み（`game_style.css`）
- ドット絵の輪郭が崩れないよう、拡大縮小は整数比ベースで調整する

### 4.4 「吸い込まれる」アニメーション（GSAP）
正解時:
```javascript
gsap.to(piece, {
    x: slot.x,
    y: slot.y,
    scaleX: 1.0,
    scaleY: 1.0,
    duration: 0.3,
    ease: "back.out(1.7)"
});
// 枠にぴったりはまった後、ぷるんと弾む
gsap.fromTo(piece.scale, { x: 1.15, y: 0.85 }, {
    x: 1, y: 1,
    duration: 0.3,
    ease: "elastic.out(1, 0.4)"
});
```

不正解時:
```javascript
// ぶるっと揺れて元の位置に戻る
gsap.to(piece, {
    x: piece.originalX,
    y: piece.originalY,
    duration: 0.4,
    ease: "power2.out"
});
```

### 4.5 レベル遷移
- 全ピースをはめ終える → 1秒待ち → 「すごい！」ボイス
- 次レベルがあれば: ステージをクリア → 新しい図形セットを配置（ポップインアニメーション）
- 最終レベルをクリアした場合: はなまる表示 → クリアオーバーレイ → シール選択

## 6. 背景画像

`games/002_shape_fit/images/background_shapes.png` として、ドット絵テイストの背景画像を生成:
- パステル系のやわらかい色合い
- 小さな幾何学模様（丸・三角・四角）が散りばめられたパターン
- ゲームプレイの邪魔にならない控えめなデザイン
- Nano Banana (Gemini) でクロマキー不要（背景は全面使用のため透過不要）

### 6.1 図形・枠アセット
背景だけでなく、**図形ピースそのもの**と**はめこみ枠**もドット絵アセットとして生成・調整する。

- 生成後に `64x64` の正方形PNGへ統一
- 余白量を揃え、見た目サイズを一致させる
- 枠はピースと同じシルエットをベースにして、色替え・影追加で制作する
- 表示時にアスペクト比を崩さず、横長・縦長の変形をかけない
- 幼児向けに判別しやすいよう、細すぎる突起や穴は避ける

## 9. 実装ステップ

1. **ディレクトリ・基本ファイル作成**: `game.json`, `start.html` のスケルトン
2. **VOICEVOX 音声生成**: 9つの音声ファイルを生成
3. **背景画像生成**: Nano Banana でドット絵テイスト背景を生成
4. **図形・枠アセット生成**: 6図形分のピース画像と枠画像をドット絵で用意
5. **サムネイル生成**: ゲーム選択画面用のサムネイル画像
6. **PixiJS ゲームコア実装** (`js/game.js`):
   - ドット絵アセットの読み込み
   - `NEAREST` スケーリング設定
   - 枠の配置ロジック
   - ドラッグ＆ドロップ機構
   - 当たり判定（ピースと枠のマッチング）
   - レベル管理ステートマシン
7. **アニメーション実装** (GSAP):
   - ピース出現アニメーション
   - 吸い込みアニメーション（正解時）
   - 揺れアニメーション（不正解時）
   - レベル遷移アニメーション
8. **音声統合** (Howler.js):
   - BGM、SE、ボイスの読み込みと再生
9. **クリアフローとシール連携**:
   - はなまる表示 → オーバーレイ → シール選択
   - `stickers.json` への登録
10. **main.js への組み込み**:
   - games 配列に追加
   - クリーンアップ処理の確認
11. **テスト作成・実行**:
    - アセットテスト
    - 既存テストの回帰確認

## 10. 注意事項

- PixiJS の `app.destroy()` は `{ children: true }` のみ（テクスチャ共有問題の回避、001で学んだ教訓）
- SPA内のアセットパスは `games/002_shape_fit/...` や `static/...` の形式（`../../` は使わない）
- `h-screen` / `100vh` は使わず `flex-1` でレイアウト
- VOICEVOX クレジット「VOICEVOX:四国めたん」を表示
- カタカナが自然な語（シール、クリア、ハート）はカタカナで表記
- ドット絵アセットは PixiJS 側で `NEAREST` 補間を使い、縮尺は整数倍を優先する
